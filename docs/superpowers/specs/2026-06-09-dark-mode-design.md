# Dark Mode — Design Spec

**Date:** 2026-06-09
**Branch:** `feat/dark-mode`
**Status:** Approved (design), implementation in progress

## Goal

Add a light/dark theme toggle to the static site `lnzh.org`. Light mode (current beige `#f2f2e4` paper / dark ink) is the default; dark mode inverts to a warm near-black `#16150F` paper with beige ink. The toggle lives fixed in the **top-right** corner as a pixel-art (8-bit) button. Switching themes plays a **pixel-dissolve** transition (blocky 8-bit checker redraw, ~500ms). The site renders many titles/icons as black PNGs, which must get beige variants so they remain visible in dark mode.

## Hard constraints discovered (must respect)

1. **CSP blocks inline scripts.** Every page `<head>` has
   `Content-Security-Policy: ... script-src 'self'` (no `'unsafe-inline'`).
   → **All JS must be external files.** No inline `<script>…</script>` anywhere.
   `style-src` *does* include `'unsafe-inline'`, so inline styles / injected `<style>` are fine.
2. **No shared layout.** 31 hand-authored `index.html` pages. All link
   `assets/css/style.css` and include `assets/js/nav-dropdown.js`. Rollout is uniform but per-file.
3. **Page-relative asset paths differ by depth:** root pages use `assets/…`,
   depth-1 (`about/`, `blog/`) use `../assets/…`, depth-2 blog posts (`blog/tomato/`) use `../../assets/…`.
   Any injected include must match the prefix the page already uses for `nav-dropdown.js`.
4. **Light mode must remain pixel-identical** after tokenization — the CSS refactor is behavior-preserving in light mode.
5. New blog pages are generated by `build.py` (gitignored) from a template — its template must also be updated so future pages inherit the theme wiring.

## Architecture

### A. Color tokenization (`assets/css/style.css`)
- `style.css` (1750 lines) currently has **zero** CSS variables and ~25 hardcoded colors.
- Extract a `:root` token set; replace hardcoded usages with `var(--token)`. Light values must equal the current literals exactly.
- Token groups:
  - **Surfaces:** `--paper` (#f2f2e4), `--paper-raised` (#fffdf4)
  - **Text:** `--ink` (strong/#000), `--text` (#333 body), `--muted` (#666), `--faint` (#999)
  - **Lines:** `--line` (#ccc / #e0e0e0), `--line-strong` (#000)
  - **Links:** `--link` (#119c36), `--link-hover` (#0a4a1a)
  - **Category/accent palette:** the ~12 tag colors (#c7873a, #b5524d, #a69a38, #7a5c8c, #4a7c8c, #3d8c57, #a16a2e, #8a3f3b, #857b2d, #5f4770, #3a6270, #2d6940) → `--cat-*`.
- **Dark overrides** in `html[data-theme="dark"] { … }`:
  - `--paper:#16150F`, `--paper-raised:#211F16`, `--ink:#F2F2E4`, `--text:#D8D6C8`, `--muted:#A8A596`, `--faint:#6F6D5F`, `--line:#3A382E`, `--line-strong:#C9C6B0`.
  - Links brightened for contrast (e.g. `--link:#3FD06A`, `--link-hover:#8FE6A8`).
  - Category colors lightened only where they fail contrast on `#16150F` (verifier checks AA).
- Add a short global `color`/`background-color` transition (e.g. 200ms) so non-dissolve repaints aren't jarring — but the dissolve overlay is the primary effect.

### B. No-flash init (`assets/js/theme-init.js` — NEW)
Tiny, **synchronous, render-blocking** script placed in `<head>` **before** the stylesheet `<link>`:
```js
(function(){try{var t=localStorage.getItem('theme');
if(t!=='light'&&t!=='dark'){t=matchMedia('(prefers-color-scheme: dark)').matches?'dark':'light';}
document.documentElement.setAttribute('data-theme',t);}catch(e){}})();
```
External (CSP-safe). Runs before first paint → no flash.

### C. Toggle + transition (`assets/js/theme-toggle.js` — NEW)
Loaded like `nav-dropdown.js` (runs on DOMContentLoaded). Responsibilities:
1. **Inject the toggle button** (fixed top-right, pixel-art styling, `role`/`aria-pressed`, keyboard operable, `prefers-reduced-motion` aware). Resolves its own asset paths via `document.currentScript.src` (mirror `nav-dropdown.js`'s URL pattern).
2. **Pixel-dissolve (DOM overlay grid):** on toggle, create a fixed full-viewport overlay containing a CSS grid of ~24px cells filled with the *outgoing* `--paper` color; flip `data-theme` underneath (page restyles instantly, hidden by overlay); clear cells in randomized checker order with staggered transition delays (~500ms total) revealing the new theme as blocks; remove overlay. `prefers-reduced-motion` → instant swap, no overlay.
3. **Persist** choice to `localStorage.theme`; update button `aria-pressed`.
4. **Swap themed images:** on init + on theme change, for every `<img>` whose src resolves to a known monochrome asset (the recolor manifest), swap to its `-dark.png` sibling in dark mode and back in light. Preload `-dark` variants once to avoid flicker.

### D. PNG beige variants (`tools/recolor_assets.py` — NEW, persistent)
- Pillow script; scans non-`instagram-*` PNGs under `assets/`. Auto-classifies (measured):
  - **Flat monochrome** (≤ ~600 unique opaque RGB; black + anti-aliasing — ~30 wordmarks/icons/arrows/stars/`hr`): recolor opaque pixels to beige `#F2F2E4`, preserve alpha. Write `<name>-dark.png`.
  - **Shaded grayscale illustrations** (clipart + coffee art, thousands of shades — ~13): **luminance-invert** (per-channel `255-v`) with a slight warm tint toward beige, preserve alpha. Write `<name>-dark.png`.
  - **Exclude** the 5 full-color assets (`cashapp/venmo/zellelogo`, `cashapp.png`, `favicon.png`) and anything under `assets/instagram-*`.
- Emits a JSON **manifest** (`tools/recolor_manifest.json`) listing source→variant + class, consumed by `theme-toggle.js` and the verifier. Idempotent and re-runnable.

### E. Rollout (`tools/inject_theme.py` — NEW, persistent)
- Idempotently edits all 31 `index.html` pages:
  - Insert `<script src="<prefix>js/theme-init.js"></script>` in `<head>` immediately before the `style.css` `<link>` (prefix derived from the page's existing `nav-dropdown.js` src).
  - Insert `<script src="<prefix>js/theme-toggle.js"></script>` next to the `nav-dropdown.js` include.
- Update `build.py`'s page template to include both for future blog pages.

### F. Demo page (`tools/theme-demo.html` or `dev/theme-demo.html` — NEW)
Standalone page animating the pixel-dissolve (and the CRT runner-up) on swatches, so the feel can be confirmed before/after site-wide wiring. Not linked from the site.

## Acceptance criteria
1. Light mode renders pixel-identically to pre-change (tokenization is behavior-preserving).
2. Dark mode: `#16150F` paper, beige ink, all text ≥ WCAG AA (4.5:1 body / 3:1 large); links & accents readable.
3. No FOUC in either mode on load or navigation.
4. All 43 monochrome PNGs have visible `-dark` variants; swapped in dark; 5 color logos untouched.
5. Pixel-dissolve plays (~500ms) on toggle; instant under `prefers-reduced-motion`.
6. Toggle top-right; keyboard accessible; `aria-pressed` correct; persists across pages & reloads.
7. Zero CSP violations (all scripts external `'self'`).
8. All JS passes `node --check`.
9. Existing nav dropdown, external-link handling, and cursor sprite still work.
10. `build.py` template updated.
